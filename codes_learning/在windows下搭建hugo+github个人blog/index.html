在Windows上搭建Hugo+GitHub个人Blog

一直觉得有必要把平时做的工作和胡思乱想的内容做成系统性的笔记，同时注重内容本质的可视化。折腾两天看了许多教程，决定选用Hugo+GitHub Page来搭建这个静态blog。那整个搭建的过程就作为第一篇笔记吧。

一、准备工具

操作系统：Windows 10 64位

教程引用：https://moyu.best/2020/01/hugogithubpages/

所需下载/安装软件：

  • Markdown编辑器： markdown本质是排版文本，更注重内容的结构和逻辑，包含基础的语法，易于上手。

    Windows上我用的是Typora , OSX上可以用Mou、MacDown等

  • Git：分布式版本控制软件，值得好好研究

  • Hugo: 一种静态网站生成器。适用于搭建个人博客、小型主页等网站。GitHub上有其他软件推荐，选择Hugo在于快速，有Windows版本，配置相对简单，支持markdown文件。

  • GitHub: 用于托管Hugo内容的远程服务器，需要注册账号，建立仓库。有需要的话可以下载GitHub Desktop 桌面客户端，也可以用git bash直接推动。

二、 使用Hugo 创建静态网站

  1. 下载符合自己laptop版本的Hugo后，新建hugo及hugo/bin文件夹，将下载的.exe解压在bin文件夹下，添加环境变量。

  2. 打开git bash，查看hugo版本

    $ hugo version

  3. 安装成功后，创建网站文件夹

    $ hugo new site C:\Hugo_site

  4. 进入新创建的网站文件夹，content\post用于存放.md文件，themes用于保存下载主题，config.toml修改配置文件

  5. git bash 执行以下命令 下载主题

    $ git clone https://github.com/pacollins/hugo-future-imperfect-slim.git

    将下载的theme内example文件下的config.toml 拷贝于Hugo_site目录下，可根据config配置文件修改模板参数。

  6. 本地运行网站确认

    $ hugo server -D

    在浏览器中打开：http://localhost:1313/ 测试页面效果，运行后修改config参数，可实时看到网页变化。直至Ctrl+C终止。

  7. 新建blog文章，以.md文件保存

    $ hugo new post\blog.md

    在typora中打开新建文件blog.md，将第三行的draft: true 修改为false。添加内容，保存后，执行6中命令运行，查看效果。至此，本地blog配置完成。

  8. 如需在md文件中插入图片可保存在Hugo_site\static\目录下，以下格式插入保存。

    ![image](/1.PNG) or ![image](/new/1.PNG)

三、配置Hugo作为一个GitHub Pages

  1. 首先，需要注册一个GitHub账号，可用于免费储存静态html，js，css，这里用于搭建个人blog。登录后注册新的仓库，命名格式 username.github.io，后面会作为网页域名。

  2. 创建完后，回到Hugo_site目录下，执行以下命令生成public目录，即GitHub仓库所需的静态blog文件

    $ hugo --theme=hugo-notepadium --baseUrl="https://hugesun1103.github.io/" --buildDrafts

  3. 进入public文件夹，将本地所有文件添加，提交到本地

    $ git init
$ git add .
$ git commit -m "First submit of my hugo blog"

  4. 将本地public推送至GitHub服务器，第一次推动需要输入GitHub账号密码

    $ git remote add origin https://github.com/hugesun1103/hugesun1103.github.io.git
$ git push -u origin master

    注意push时出现报错，修改命令后推送成功。刷新GitHub页面，public文件上传，在浏览器中输入 https://hugesun1103.github.io/ 即可访问。

    $ git push -f origin master

  5. 日常更新，在修改好md文件后，需要执行下述命令编译后查看效果。

    $ hugo
$ hugo server -D

测试效果没问题后，运行下述sh deploy.sh脚本，自动部署。

#/bin/bash
echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

msg="rebuilding site `date`"
if [ $# -eq 1  ]
    then msg="$1"
fi

# Go To Public folder
cd public

# Add changes to git.
git add -A

# Commit changes.

git commit -m "$msg"

# Push source and build repos.
git push origin master

cd ../
